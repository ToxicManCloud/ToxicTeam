import time
import random
import matplotlib.pyplot as plt
%matplotlib inline

def bubble(nums):
    swapped = True
    while swapped:
        swapped = False
        for i in range(len(nums) - 1):
            if nums[i] > nums[i + 1]:
                nums[i], nums[i + 1] = nums[i + 1], nums[i]
                swapped = True

def insertionsort(nums):
    for j in range(1, len(nums)):
        key = nums[j]
        i = j - 1
        while i >= 0 and nums[i] > key:
            nums[i + 1] = nums[i]
            i -= 1
        nums[i + 1] = key

def bubble_sort(nums):
    n = len(nums)
    for i in range(n):
        for j in range(0, n-i-1):
            if nums[j] > nums[j+1]:
                nums[j], nums[j+1] = nums[j+1], nums[j]

1. Нотація в контексті алгоритмів та комп'ютерної науки використовується для опису складності алгоритмів, їхньої швидкості та ресурсів, які вони вимагають. Нотація може бути математичними виразами, символами чи термінами, що описують час, пам'ять або інші аспекти роботи алгоритму.
Різниця між нотацією і нотацією:
Нотація - це загальний термін, який використовується для опису різних концепцій та ідей, включаючи термінологію, символи та математичні вирази.
Нотація складності алгоритмів (наприклад, "O-нотація", "Ω-нотація", "Θ-нотація") - це конкретні методи оцінки ефективності алгоритмів, які використовують математичні вирази та символи для вираження рівня складності.

2. Часова складність алгоритму сортування методом вставки в найсприятливішій умові (коли масив вже відсортований):

3. У найсприятливішій умові, коли масив вже відсортований, часова складність алгоритму сортування методом вставки буде O(n), де n - кількість елементів у масиві. Це тому, що в такому випадку алгоритм просто перевіряє кожен елемент і не потребує переміщення жодного елементу, оскільки вони вже знаходяться на своїх місцях.
Ефективний алгоритм - це алгоритм, який забезпечує оптимальне використання ресурсів (таких як час та пам'ять) для вирішення певної задачі. Ефективний алгоритм може мати мінімальну часову складність, мінімальний обсяг використовуваної пам'яті або інші оптимальні характеристики залежно від поставленої задачі.

4.Головний параметр (розмір) задачі - це кількість вхідних даних, яка визначає складність алгоритму. У контексті алгоритмів головний параметр може бути кількість елементів у масиві, розмір вхідного тексту або будь-яка інша характеристика вхідних даних, яка впливає на роботу алгоритму. Часто головний параметр використовується для оцінки часової та просторової складності алгоритму.
